## Clean Code Instructions

### Naming Conventions
   1. Use meaningful and descriptive names for variables, functions, classes, and modules.
   2. Follow PSR-12 and Laravel naming conventions (e.g., camelCase for variables/methods, StudlyCase for classes, snake_case for database columns).
   3. Boolean variables should be prefixed with `is`, `has`, or `can` (e.g., `$isRegistered`, `$hasCompleted`).

### PHP & Laravel Best Practices
   1. Use PHP 8 constructor property promotion for dependency injection.
   2. Always use explicit return type declarations and type hints for method parameters.
   3. Use curly braces for all control structures.
   4. Prefer PHPDoc blocks over inline comments for complex logic.
   5. Use Eloquent models and relationships instead of raw DB queries whenever possible.
